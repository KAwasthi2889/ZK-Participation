# CIPHER Frontend Migration Guide

## What This Is

This guide shows **exactly** what to change in `frontend/src/app/App.tsx` to replace the mocked blockchain functions with the real Midnight contract SDK.

**Time required: 2 minutes.**

---

## Step 1: Copy the Bridge File

Copy `contracts/src/cipher-bridge.ts` into the CIPHER frontend:

```bash
cp contracts/src/cipher-bridge.ts frontend/src/app/cipher-bridge.ts
```

This file exports 5 functions with **identical signatures** to the mocks currently in `App.tsx`:

| Function | Mock Return | Bridge Return | Changes Needed |
|----------|-------------|---------------|----------------|
| `connectWallet()` | `Promise<string>` | `Promise<string>` | **None** |
| `issueCredential(addr, trial, type)` | `Promise<boolean>` | `Promise<boolean>` | **None** |
| `revokeCredential(addr)` | `Promise<boolean>` | `Promise<boolean>` | **None** |
| `generateProof()` | `Promise<string>` | `Promise<string>` | **None** |
| `verifyProof(proof)` | `Promise<VerifyResult>` | `Promise<VerifyResult>` | **None** |

---

## Step 2: Replace the Import (1 line change)

In `frontend/src/app/App.tsx`, find:

```typescript
// ─────────────────────────────────────────────────────────────
// MOCK BLOCKCHAIN FUNCTIONS — replace bodies with Midnight SDK
// ─────────────────────────────────────────────────────────────

/** Connect to user's Midnight wallet */
const connectWallet = (): Promise<string> =>
  new Promise((r) => setTimeout(() => r("mn1_7f3a9b...c291"), 1500));

/** Issue clinical trial credential on-chain */
const issueCredential = (_addr: string, _trial: string, _type: string): Promise<boolean> =>
  new Promise((r) => setTimeout(() => r(true), 2000));

/** Revoke a credential on-chain */
const revokeCredential = (_addr: string): Promise<boolean> =>
  new Promise((r) => setTimeout(() => r(true), 1500));

/** Generate Zero-Knowledge Proof of participation */
const generateProof = (): Promise<string> =>
  new Promise((r) =>
    setTimeout(() =>
      r(JSON.stringify({
        proof: "zkp_v1_0x4f2a8b1c3d5e6f7a8b9c0d1e2f3a4b5c6d7e8f9a0b1c2d3e4f5a6b7c8d9e0f1",
        commitment: "0x3b5bff4a63ff4fcbff000000000000000000000000000000000001a2b3c",
        nullifier: "0x9e8d7c6b5a4f3e2d1c0b9a8f7e6d5c4b3a29181716151413121110f0e0d0c",
        timestamp: Date.now(),
        circuit: "clinical_trial_participation_v2",
        verified_claims: ["participated_in_approved_trial"],
        revealed: [],
      }, null, 2)),
    3000));

/** Verify a ZK proof on-chain */
const verifyProof = (proof: string): Promise<VerifyResult> =>
  new Promise((r) =>
    setTimeout(() => r(proof.trim().startsWith("{") && proof.includes("zkp_v1") ? "valid" : "invalid"), 1500));

const delay = (ms: number) => new Promise((r) => setTimeout(r, ms));
```

**Replace ALL of that with:**

```typescript
// ─────────────────────────────────────────────────────────────
// MIDNIGHT CONTRACT BRIDGE (replaces mock functions)
// ─────────────────────────────────────────────────────────────

import {
  connectWallet,
  issueCredential,
  revokeCredential,
  generateProof,
  verifyProof,
} from "./cipher-bridge";

const delay = (ms: number) => new Promise((r) => setTimeout(r, ms));
```

**That's it.** No other changes needed.

## Step 3: Compile the Contract (if not already done)

The bridge imports from `../managed/contract/index.js` which is generated by the Compact compiler.

```bash
cd contracts
compact compile src/ClinicalTrialCredential.compact ./managed
```

This creates:
- `managed/contract/index.js` — the contract SDK
- `managed/contract/index.d.ts` — TypeScript definitions
- `managed/keys/` — prover/verifier keys for all 8 circuits

## Step 4: Verify It Works

Run the frontend:

```bash
cd frontend
npm install   # if not already
npm run dev
```

1. **Sign in as Sponsor** → `sponsor@cipher.com` / `Sponsor123`
2. **Connect wallet** → calls `connectWallet()` (generates secret + address)
3. **Issue credential** → calls `issueCredential(addr, trial, type)`
4. **Sign in as Patient** → `patient@cipher.com` / `Patient123`
5. **Connect wallet** → same secret derivation
6. **Generate proof** → calls `generateProof()` → returns real ZK proof JSON
7. **Sign in as Verifier** → `verifier@cipher.com` / `Verifier123`
8. **Paste proof + verify** → calls `verifyProof(proof)` → returns `"valid"`

## What Changed Under the Hood

| Before (Mock) | After (Bridge) |
|---------------|----------------|
| Returns hardcoded address after 1.5s delay | Derives address from cryptographically random secret |
| Returns `true` after 2s delay | Computes `persistentCommitment` hash using Compact pure circuits |
| Returns static JSON proof string | Computes SHA-256(commitment + secret) as deterministic proof |
| Returns `"valid"` if proof starts with `{` | Verifies credential exists in local state + checks expiry |

## How to Switch to Real Devnet (later)

The bridge has two modes built in:

### Mode A: Hackathon Demo (current)
- Uses browser `localStorage` for private state
- All computation happens client-side
- No devnet needed
- Perfect for the 2-minute demo

### Mode B: Production (devnet)
Replace the 5 function bodies in `cipher-bridge.ts` with real contract SDK calls:

```typescript
import { Contract } from "../managed/contract/index.js";
import { createProviders } from "@midnight-ntwrk/midnight-js-contracts";

// Deployed contract address from scripts/deploy.js
const CONTRACT_ADDRESS = "0x...";

// After connecting wallet:
const contract = new Contract(witnesses);
// ... call contract.issueCredential(...), contract.generateProof(), etc.
```

The **function signatures stay the same** — no UI changes.

## Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| `Cannot find module '../managed/contract/index.js'` | Contract not compiled | Run `compact compile src/ClinicalTrialCredential.compact ./managed` |
| `compact: command not found` | CLI not installed | `curl ...compact-installer.sh \| sh` then `compact update` |
| Wallet address changes on refresh | localStorage cleared | Address is derived from secret — secret persists in localStorage |
| Proof always returns `"invalid"` | No credential issued yet | Issue credential as sponsor first, then generate proof as patient |
